 JPTインターン 設計相談の引き継ぎメモ


## 決定済み事項

### 技術スタック
- **バックエンド：** Laravel + Breeze (Inertia + React)
- **フロントエンド：** React + Tailwind CSS
- **DB：** MySQL
- **デプロイ先：** Laravel Cloud（ハイバネーション復帰約10秒、設定で調整可能）
- **開発ツール：** Claude, Cursor（全て課金プラン）

### 画面ナビゲーション構成
サイドバーで以下の5つに分ける：
1. ダッシュボード（ホーム）
2. 案件管理（タスク管理含む）← メインで使う画面
3. 承認フロー ← 管理者が時々使う
4. 予算管理 ← 確認用、頻度低め
5. 通知

### ロールと権限
- **申請者（applicant）：** 案件申請、自案件の進捗・予算入力、ステータス確認
- **部門管理者（dept_manager）：** 一次承認/却下、自部門の案件・進捗確認
- **本部管理者（hq_manager）：** 最終承認/却下、全案件の一覧・進捗・予算確認

### タスク管理の方針
- Backlog風にする（種類・優先度・カテゴリ・コメント・変更履歴）
- PoCではリスト表示＋詳細画面。カンバン・ガントは将来対応
- DBは将来拡張を見据えて整えておく（ポートフォリオとして継続開発予定）

### 課題2（+α）の方針 モックのみ実装
- ダッシュボードで全案件の進捗・予算状況を可視化（グラフ・チャート）
- 承認ステップの可視化（ステータスバー）
- 部門管理者向けメンバータスク状況一覧

### 承認フローの設計判断
- 却下→再申請は新規レコードにコピー（parent_project_idで元案件に紐づけ）
- 部門管理者が申請した場合は部門承認スキップ（自己承認防止）
- 承認履歴はapprovalsテーブルで管理（監査証跡）

### 予算管理
- projectsテーブルにbudget_amount / actual_amountを直接持たせる（シンプル方式）
- 費目別管理は将来budget_itemsテーブル追加で対応
