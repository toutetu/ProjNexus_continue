# V0 プロンプト集 - 開発管理統合アプリケーション PoC

> V0 (v0.dev) で画面を作るためのコピペ用プロンプト集。
> **最初に「共通プロンプト」を1度送ってから**、各画面プロンプトを個別に送ってください。
>
> **現行ワークフロー**: Claude で HTML モック → Cursor で Inertia/React 実装 が主軸。
> 本ドキュメントは V0 を使う場合の保険・参考用として整備しています。
>
> **スコープ（2026-04-16）**: 課題1実装必須のものを「★ 必須」、課題2扱いで後回しのものを「☆ 後回し」でマーク。

## 0. 共通プロンプト（最初に1度だけ送信）

```
これから日本企業向けの開発管理統合Webアプリ（PoC）の画面を複数作成します。
すべての画面で以下のデザイン仕様を厳守してください。

# 技術
React + TypeScript + Tailwind CSS + shadcn/ui
アイコン: lucide-react
グラフ: Recharts
日本語UI

# フォント
Noto Sans JP（日本語）+ Roboto（英数字）

# カラー
- プライマリCTA: #E60013（JGCレッド）— 申請・承認・保存ボタン
- テキスト濃: #212429
- ページ背景: #F8F9FA
- カード背景: #FFFFFF
- 補助テキスト: #6C757D
- ボーダー: #DEE2E6
- 情報系/リンク: #106EBE
- アクセント: #EDB100

# ステータスバッジ色（必ず統一）
- draft（下書き）: グレー #6C757D
- pending_dept（部門承認待ち）: シアン #01CFFF
- pending_hq（本部承認待ち）: ブルー #106EBE
- approved（承認済）: グリーン #16A34A
- rejected（却下）: レッド #E60013
バッジは「色付き丸 + アイコン + 日本語ラベル」の3点セットで、rounded-full px-3 py-1 text-xs font-medium。

# レイアウト共通
- ヘッダー: h-14, 白背景, border-bottom, 左にロゴ「JPT 開発管理」、右に「⌘K」検索バッジ + 通知ベル(バッジ付き) + ユーザーアバター
- サイドバー: w-64, bg #212429（ダーク）, 白文字。アクティブ項目は左端にJGCレッド(#E60013)の縦バー4px
- サイドバーメニュー: 3セクションのグルーピング
  ・セクション「申請・承認」: 新規申請 / 承認待ち一覧 / 案件一覧（申請タブ）
  ・セクション「開発管理」: 案件一覧（開発タブ） / タスク一覧(課題2扱い)
  ・セクション「予算管理」: 案件一覧（予算タブ） / 予算実績入力導線
  ・セクション下部（共通）: 通知 / プロフィール
  セクション見出しは小さめの uppercase(text-xs, tracking-wider, text-gray-400)で区切り、各項目はインデント
- メインコンテンツ: bg #F8F9FA, max-w-7xl, p-6
- レスポンシブ: モバイルでサイドバーをドロワー化

# インタラクション
- ボタン: rounded-md, shadow-sm hover:shadow-md, active:scale-95, transition-all duration-150
- カード: rounded-lg border shadow-sm。クリック可能なら hover:-translate-y-0.5 hover:shadow-md
- テーブル行（クリック可能）: hover:bg-gray-50 cursor-pointer
- フォーカスリング: focus:ring-2 focus:ring-blue-500
- disabled: opacity-50 cursor-not-allowed

# UX要素
- 空状態（Empty State）: アイコン64px(jpt-muted) + メインテキスト + サブテキスト + CTAボタン
- ローディング: スケルトン（bg-gray-200 animate-pulse）
- 成功/エラー通知: 右上トースト（sonner）
- 進捗バー: h-2 rounded-full、0-50%緑/50-70%黄/70%超赤
- コマンドパレット: ヘッダー右に「⌘K」バッジ。Cmd+K / Ctrl+K で起動できる shadcn/ui の Command コンポーネント
- パンくずリスト: ページ上部に常設（ホーム > 親ページ > 現在のページ）

# ログイン直後のホーム
ダッシュボードは課題2扱いで後回しのため、ログイン直後は「案件一覧（申請タブ）」を暫定ホームとする。

了解したら「了解しました」とだけ返してください。次のメッセージで最初の画面を依頼します。
```

```

- 申請→部門承認→本部承認の順で表示

【変更履歴タブ】
```

## 使い方のコツ

1. **共通プロンプトを必ず最初に1度送る** → V0が前提を理解した状態で各画面を作れる
2. **1画面1プロンプト**で送る → V0は1メッセージで複数画面を作るのが苦手
3. **「もう少し○○して」と修正指示を続けて出す** → 最初は完璧でなくてよい
4. 各画面ができたら **コードをコピーして Cursor に持ち込む**
5. **S-03 (a/b/c) は同一Reactコンポーネントで tab props を切替** → V0で3バリアント作らせたあと1本に統合すると楽
6. ロール別バリアントは「S-02を○○用にして」と差分指示でOK
7. **課題1優先**: ★マークの画面から着手。☆マークはプレゼン用モックのみで実装スキップ
