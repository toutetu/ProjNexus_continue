# S-03c 案件一覧（予算タブ） 方針

## 構成

- **背景**: 承認済み案件の予算消費状況を俯瞰する画面。サイドバー「予算管理 > 案件一覧」から遷移
- **レイアウト**: AuthenticatedLayout（サイドバー「予算管理 > 案件一覧」がアクティブ）、max-w-7xl
- **パンくず**: 予算管理 > 案件一覧
- **URL**: `/projects?tab=budget`
- **コンポーネント**: S-03a / S-03b と同じ `Projects/Index.tsx` を tab props で列切替

## 構成セクション（上から下へ）

### ページヘッダー
- タイトル左に `.title-accent`
- 「案件一覧」+ サブタイトル「予算フェーズ · 全5件」
- 右端に「新規申請」ボタン（統一のため配置）

### タブ切替UI（budget がアクティブ）
- 3タブ共通。budget タブの件数バッジが赤背景で強調

### サマリーカード 4枚（grid: md=2列, lg=4列、タブ直下）
1. **予算合計**（blue アクセントライン）: 確定予算の総和、mono 大文字
2. **実績合計**（cyan アクセントライン）: 実績額の総和
3. **平均消費率**（purple アクセントライン）: 消費率の平均 + プログレスバー
4. **要注意**（赤アクセントライン）: 消費率 85% 超の件数、赤数字 + 「件」

※ カード計算はロール別に絞り込んだ母集団で計算（hq_manager なら全社、dept_manager なら自部門）

### フィルタバー
- 検索ボックス: 案件名で検索
- 部門セレクト: すべて / 開発1部 / 開発2部 / 開発3部
- 消費率セレクト（予算タブ固有）: すべて / 60%未満 / 60-85% / 86-100% / 超過
- 右端に「クリア」リンク

### テーブル（ProjectTableBudgetTab）
- 列構成: タイトル / 部門 / 予算額 / 実績額 / 消費率 / 更新日 / chevron
- **タイトル列**: 案件名 + PRJ-ID（S-03b と同形式）
- **予算額・実績額**: 右寄せ mono（例: `¥10,000,000`）。JPY はカンマ区切り、先頭に `¥` 固定
- **消費率列**:
  - プログレスバー（w-32, h-2）+ パーセンテージ（右側 mono w-12）
  - バー色分け（4段階）:
    - `.cr-safe` 0-60%: 緑 `#16A34A`（余裕）
    - `.cr-normal` 61-85%: 青 `var(--jpt-blue)`（順調）
    - `.cr-warn` 86-100%: 橙 `#F59E0B`（注意）
    - `.cr-over` >100%: 赤 `var(--jpt-red)`（超過）
  - 90% 超にはバー右端に alert-triangle アイコンを重ねて警告マーカー
- **更新日**: 最後に `budget_actuals` が追加された日時
- 行クリック → S-04 案件詳細の「予算サマリー」セクションへスクロール、または S-11 予算実績入力モーダルを直接開く（実装時に決定）

### ページネーション
- S-03a / S-03b と同じ構成

## ロール別データアクセス

| ロール | 表示される案件 |
|---|---|
| applicant | `status=approved` かつ自案件（主担当） |
| dept_manager | `status=approved` かつ `department_id = self.dept` |
| hq_manager | `status=approved` の全件 |

## 設計上のこだわり

- **サマリーカード4枚で「全体感」を最優先で伝える** — 予算管理の利用者（主に本部管理者）は「全体でどのくらい使っているか」「何件注意が必要か」を最初に知りたい。テーブルに入る前にこの4数字で全社状況をつかめるようにした
- **「要注意」カードで赤バッジ件数を最上段に配置** — 予算超過リスクは最も早く気づくべき情報。4枚の中で赤を使うのは「要注意」だけに限定して、視覚的に最優先であることを示す
- **消費率バーの4段階色分け（S-11 と連動）** — S-11 予算実績入力モーダルの閾値ガイダンス（86% で注意・100% で超過）と同じしきい値を使う。画面間で色とルールが一致することで、ユーザーは「86% を超えたら色が変わる」という一貫した挙動を期待できる
- **実績額のみを mono で強調せず、予算額も mono で併記** — 両方 mono にすることで桁を揃えて視覚的な比較がしやすい。「予算との差分」を暗算しやすくする
- **消費率フィルタを用意** — 「超過だけ見たい」「安全圏だけ見たい」など、運用者の関心軸で絞れるようにする。ダッシュボードと違い、一覧画面は「目的を持って探しに来る場所」なのでフィルタ軸は細かく提供
- **90% 超で警告マーカー**（alert-triangle アイコン） — バーの色変化だけでなく、行単位でアイコンを添えることで「注意すべき行」を走り読みでも見逃さない冗長化
- **PRJ-ID を一覧で表示** — 予算会議などで「PRJ-2026-0042 の実績は？」と参照される運用を支援
- **消費率は DB に持たず集計** — `projects.actual_amount / projects.budget_amount * 100` で算出。冗長カラムを作らず、常に最新の比率を表示する
