# S-02 ダッシュボード 方針

> **位置づけ**: 課題2（+α機能）。PoC スコープでは **実装しない**。プレゼン資料と将来拡張のために設計のみ残す。React 実装する場合の指針としても流用可能。

## 構成

- **背景**: 本部管理者の入口画面。全社横断の状況を俯瞰するための経営ダッシュボード
- **レイアウト**: AuthenticatedLayout（サイドバー「ダッシュボード」がアクティブ — PoCでは項目なし）、max-w-7xl
- **パンくず**: なし（ホーム画面扱い）
- **URL**: `/dashboard`
- **アクセス**: `hq_manager` のみ。他ロールは `/projects?tab=approval` にリダイレクト
- **ホーム遷移の扱い**: 課題1では本部管理者も `/projects?tab=approval` を暫定ホームに。本画面は課題2で解禁

## 構成セクション（上から下へ）

### ページヘッダー
- 「ダッシュボード」タイトル + サブテキスト「2026年4月15日（水） 全部門の状況」
- 右端に「最終更新: 14:32」（muted、取得時刻表示）
- ※ `.title-accent` の縦バーは **付けない**（情報密度が高い画面のため装飾を抑制）

### 1段目: KPI カード 4枚（grid: md=2列, lg=4列）
1. **進行中案件**（cyan→blue グラデーションライン + 数字 + ミニスパークライン + 前週比）
2. **承認待ち**（黄アクセントライン + 数字 + 「要対応」バッジ + 「承認待ち一覧へ」リンク）
3. **平均進捗率**（blue→purple グラデーションライン + パーセンテージ + プログレスバー + 目標差分）
4. **予算消費率**（赤枠 `border-jpt-red/20` + 赤アイコン + 赤数字 + pulse バー + 「警告ライン 70% を超過」）

KPI データソース:
- 進行中案件: `projects.where(status=approved)` かつ `tasks.progress < 100` を集計
- 承認待ち: `projects.whereIn(status, [pending_dept, pending_hq])` の合計
- 平均進捗率: 承認済案件の `tasks.progress_rate` の平均
- 予算消費率: 全案件の `sum(actual_amount) / sum(budget_amount) * 100`

### 2段目: グラフ 2つ（grid: lg=2列）
- **部門別進捗状況**（Recharts BarChart）
  - X軸: 開発1部 / 開発2部 / 開発3部
  - Y軸: 平均進捗率（0-100%）
  - バー色: cyan / blue / purple（部門の識別色）
- **月次予算消費推移**（Recharts LineChart）
  - X軸: 直近4ヶ月
  - Y軸: 消費率（0-100%）
  - 70% に警告ライン（赤破線）をカスタムプラグインで描画
  - ラインカラー: jpt-red + 薄い塗り

### 3段目: 予算消費 70% 超の案件リスト
- ヘッダー: alert-triangle アイコン（赤） + 「予算消費 70% 超の案件」 + 「全件を見る →」リンク
- テーブル列: 案件名 / 部門 / 予算（右寄せ mono） / 実績（右寄せ mono） / 消費率バー + % / 操作（詳細リンク）
- 消費率 80% 以上は pulse-warn アニメーションでさらに強調
- 行クリックで S-04 案件詳細へ

### フッター
- 「開発管理統合アプリ PoC | 日揮パラレルテクノロジーズ インターン 高橋朋子」（中央、muted、text-xs）

## ロール別データアクセス

| ロール | 画面アクセス |
|---|---|
| applicant | ×（`/projects?tab=approval` へリダイレクト） |

## 設計上のこだわり

- **上から下へ粒度を細かくする情報階層** — KPI（全体数値）→ トレンド（時系列・部門別）→ 個別案件（アクション対象）の順で配置。「概要をつかむ → 気づく → 行動する」の視線誘導を 1画面で完結させる
- **警告は予算消費率カードのみに集中** — 4枚のKPIのうち赤枠 + pulse アニメは1枚だけ。警告要素を増やすと麻痺するため、「最も深刻な指標」だけを強調する視覚階層を作る
- **KPIカード上部のグラデーションライン** — cyan→blue / 黄 / blue→purple / 赤 の4色でカードを識別。スクロールしたり視線が移動しても色でランドマーク化でき、再認知コストが低い
- **70% の早期警戒ライン** — S-11 の運用ルール（86% で注意・100% で超過）より手前に設置。ダッシュボードは「気づき」のためのもので、アクション画面（S-11）より早く警告を出すことで対応時間を確保する
- **ミニスパークライン** — 「進行中案件」カードの下部に小さな折れ線。数字だけでは分からない「増えているか減っているか」の変化を 8px の高さで表現する
- **70% 超リストをダッシュボード内に含める** — 別画面にせず同じページ内に置くことで、KPI で気づいた瞬間に対象案件へジャンプできる。ワンクリックで詳細に飛べる導線を短くする
- **Recharts で統一** — shadcn/ui のエコシステムと親和性が高い。Chart.js は PoC モックで使用したが、React 実装では Recharts に統一してコンポーネントの宣言的記法を維持する
