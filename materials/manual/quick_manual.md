# 利用マニュアル（簡易版）

> 各ロールが「最低限の業務を一周」できるクイックリファレンスです。
> 印刷すると **ロールごとに A4 1 枚 × 3 ページ** に分かれます。
> 詳しい操作・状態遷移・FAQ は [▼ 詳細版マニュアルを見る](#detailed) を参照してください。

## 0. 共通

### ログイン

ブラウザで `/login` を開き、メールアドレスとパスワードでログインします。

| メールアドレス | 氏名 | 部門 | ロール |
|---|---|---|---|
| `applicant@example.com` | 高橋 朋子 | 開発1部 | 申請者 |
| `dept@example.com` | 夏目 拓也 | 開発1部 | 部門管理者 |
| `hq@example.com` | 本部 一郎 | 本部 | 本部管理者 |

> パスワードは全員 `password`。フル一覧は詳細版 §3.1 参照。

### 画面の基本構成

```
+----------------------------------------+
| ヘッダー（ロゴ / 通知ベル / プロフィール）|
+--------+-------------------------------+
| サイド | メインコンテンツ              |
| バー   |                               |
|        |                               |
| 申請・ |                               |
| 承認   |                               |
| ─────  |                               |
| 開発   |                               |
| 管理   |                               |
| ─────  |                               |
| 予算   |                               |
| 管理   |                               |
+--------+-------------------------------+
```

サイドバーは **申請・承認 / 開発管理 / 予算管理** の 3 セクション構成。通知はヘッダー右上のベルから。

### 5 つの状態と承認フロー

```
[下書き] ─申請─▶ [部門承認待ち] ─承認─▶ [本部承認待ち] ─承認─▶ [承認済]
                       │                       │
                       └──── 却下 ─────┬──────┘
                                       ▼
                                    [却下] ─再申請─▶ 新案件
```

部門管理者が自分で起案した場合は **本部直行**（部門承認をスキップ）。

### ロール早見表

| 主操作 | 申請者 | 部門管理者 | 本部管理者 |
|---|:---:|:---:|:---:|
| 新規案件の申請 | ○ | ○（本部直行） | × |
| 部門承認・部門却下 | × | ○（自部門） | × |
| 本部承認・本部却下 | × | × | ○（全件） |
| タスクの作成・編集 | ○（自案件） | ○（自部門） | **閲覧のみ** |
| 予算実績の入力 | ○（自案件） | ○（自部門） | 閲覧のみ |

詳細は [ロール別機能マトリクス](https://github.com/toutetu/ProjNexus/blob/main/materials/Design/role_feature_matrix_.md) または詳細版 §2.3 を参照。

## 1. 申請者向け（クイック）

### できる／できない（抜粋）

| 操作 | 可否 |
|---|:---:|
| 新規案件の申請・下書き・編集 | ○ |
| 申請の取り戻し（承認前のみ） | ○ |
| 部門／本部の承認・却下 | × |
| 承認済かつ自分が主担当のタスク作成・予算実績入力 | ○ |

### 1-1. 新規案件を申請する

1. サイドバー **申請・承認 ＞ 新規申請** をクリック
2. 案件名・担当部門・目的・概算予算を入力（下書きなら案件名のみで保存可）
3. **下書き保存** か **申請する** を押す
4. 確認モーダルで遷移先（部門承認待ち / 本部承認待ち）を確認

### 1-2. 取り戻して下書きに戻す

承認・却下が **一度も行われていない** 場合のみ可能。案件詳細右上の **「取り戻して下書きに戻す」** ボタンから。

### 1-3. 却下されたら再申請する

1. 案件詳細上部の赤いボックスで却下コメントを確認
2. 右上の **編集** → 内容を修正 → **更新して申請**
3. 新しい案件レコードが作成され、元案件は「却下」のまま履歴に残る

### 1-4. タスクの作成・進捗更新

承認済の案件のみ可能。

1. 案件詳細の **タスク** タブ右上の **タスク追加**
2. タスク一覧（サイドバー **開発管理 ＞ タスク一覧**）の **カンバンビュー** でドラッグ＆ドロップでステータス更新
3. 自分が担当 or 確認者のタスクは編集可能

### 1-5. 予算実績を入力する

承認済かつ自分が主担当の場合のみ。

1. 案件詳細の **予算** タブ ＞ **実績を入力**
2. 実績額（円）を入力 → **上書き方式** で保存

### 1-6. 通知を確認する

ヘッダー右上の **🔔 通知ベル** から、または `/notifications`。

## 2. 部門管理者向け（クイック）

### できる／できない（抜粋）

| 操作 | 可否 |
|---|:---:|
| 自部門の案件の **部門承認・部門却下** | ○ |
| 自部門の承認済案件のタスク作成・編集 | ○ |
| 自部門の承認済案件の予算実績入力 | ○ |
| 本部承認・本部却下 | × |
| 他部門の案件の閲覧 | × |
| 自分で起案した案件は **本部直行** | ○ |

### 2-1. 自部門の承認待ちを確認する

サイドバー **申請・承認 ＞ 承認待ち** をクリック → 自分が対応すべき自部門の部門承認待ち案件が一覧される。

### 2-2. 案件を承認・却下する

1. 一覧から該当案件を開く（タイトル「承認画面」）
2. 申請内容を確認
3. 画面下部の **承認** または **却下** ボタン → コメント（任意）→ 確定
4. 承認すると **本部承認待ち** に進む

> 承認・却下は **自部門かつ部門承認待ち** の案件にのみ実行可。

### 2-3. 自分で起案した場合（本部直行）

部門管理者が自分で申請すると、部門承認をスキップして直接 **本部承認待ち** になります。一覧画面では **「本部直行」バッジ** が表示されます。本部承認前であれば取り戻しも可能です。

### 2-4. タスク一覧でメンバー進捗をレビュー

サイドバー **開発管理 ＞ タスク一覧**（`/member-tasks`）。

- **メンバー別ビュー（初期表示）**：部門メンバー × タスクのマトリクス。期限超過・確認待ちが誰に集まっているか俯瞰
- **カンバンビュー**：ステータス別の偏りを確認、DnD でステータス更新可能
- **一覧ビュー**：表形式で検索・絞り込み

### 2-5. 自部門の予算実績を入力する

承認済かつ自部門の案件は、申請者でなくても **実績を入力** ボタンから入力可能。

## 3. 本部管理者向け（クイック）

### できる／できない（抜粋）

| 操作 | 可否 |
|---|:---:|
| **本部承認・本部却下**（全件） | ○ |
| 全部門・全案件の閲覧 | ○ |
| 案件の起案・申請 | × |
| 部門承認 | × |
| **タスクは閲覧のみ**（作成・編集・DnD 不可） | 閲覧のみ |
| **予算実績の入力** | 閲覧のみ |

> タスク・予算実績の入力は方針として行いません。閲覧で全社状況を俯瞰する役割。

### 3-1. 全社を俯瞰する

サイドバー **申請・承認 ＞ 案件一覧** で 3 タブを切替。

| タブ | 内容 |
|---|---|
| 申請 | 全社の申請状況。本部承認待ちで絞り込める |
| 開発 | 承認済案件のタスク進捗。期限超過・完了間近を確認 |
| 予算 | 承認済案件の予算消費状況。上部にサマリ（合計・平均消費率・要注意件数） |

### 3-2. 最終承認・最終却下する

1. サイドバー **申請・承認 ＞ 承認待ち** → 本部承認待ち案件を開く
2. 申請内容と部門承認コメントを確認
3. 画面下部の **承認** または **却下** → コメント → 確定

承認すると：

- ステータスが **承認済** になる
- `budget_amount` に概算予算が確定値として書き込まれる
- 初期タスク **「実装計画作成」** が自動生成される
- 申請者に通知が届く

### 3-3. 開発・予算の進捗を監視する

- **開発タブ** の期限列：期限超過＝赤、2 週間以内＝赤、1 ヶ月以内＝橙
- **予算タブ** の消費率列：100% 超＝赤、86〜100%＝黄、60〜85%＝青、60% 未満＝緑

### 3-4. タスク一覧（閲覧のみ）

サイドバー **開発管理 ＞ タスク一覧**。本部管理者は **上部の「部門」を先に選択** してからカンバン／メンバー別／一覧を切り替えます。**書き込みは行わない方針**（保存・DnD・コメント投稿）。
