# S-11 予算実績入力モーダル 方針

> **実装方式の位置づけ**  
> - **課題1（PoC・実装対象）**: `projects.actual_amount` の **上書き方式**。実績額（累計）を直接入力するシンプル版  
> - **課題2（将来拡張・モック用）**: `budget_actuals` テーブルへの **追加方式**。 本ファイルで詳細設計する UI は課題2 相当で、モック画像・プレゼン資料の素材として残す  
> - **モックアップ（`s11_budget_actual_modal.html`）は課題2 版のフル仕様で作成済み**。課題1 実装時は下記「§課題1 実装版」の簡易仕様に従う

## 構成

- **背景**: 案件の予算実績（支出額）を入力するモーダル。 S-03c 予算タブの行クリック、または S-04 案件詳細の「予算実績入力」カードから開く
- **レイアウト**: 画面中央に配置するモーダル、max-w-2xl（約 640px）、背景 dimmed（`bg-black/50`）
- **パンくず**: なし（モーダル）
- **URL**: モーダル表示中は URL は変わらない
- **コンポーネント**: `BudgetActualDialog`（shadcn/ui Dialog ベース）

## モーダル構成（上から下へ）

### ヘッダー
- 左: アイコン（coins、muted背景の円）
- タイトル: 「予算実績入力」+ `.title-accent` の縦バー
- サブテキスト: `PRJ-2026-0042 / 次世代EAMシステム開発`（案件コンテキスト）
- **主担当バッジ**（右側）: avatar + 「主担当: 高橋 朋子」— 誰がこの入力を行えるかの視覚化
- 右端: × ボタン

### 現状サマリー（4カード、grid-cols-4）
- 上段に常時表示、編集前の状態を即把握
- **① 確定予算**: `budget_amount`、mono 大文字、`¥10,000,000`
- **② 実績合計**: 現時点の `actual_amount`、mono、`¥4,200,000`
- **③ 残額**: `budget_amount - actual_amount`、mono、`¥5,800,000`
- **④ 消費率**: パーセンテージ + プログレスバー（色分けは S-03c と同じ4段階）

### 入力フォーム

#### 1. 今回の支出額 *（必須）
- AmountInput（右寄せ mono、自動カンマ区切り、右端に「円」単位）
- 例: `¥1,580,000`
- バリデーション: 正の整数、0 より大きい

#### 2. カテゴリチップ選択 *（必須、4択）
- `外注費`（青） / `ライセンス`（紫） / `機材費`（緑） / `その他`（灰）
- S-10 と同じチップUI。選択時に背景色がカテゴリ別に変わる
- バックエンドは `budget_actuals.category` に enum で保存

#### 3. 用途 *（必須）
- テキスト入力、maxlength=100、placeholder「例：IoTゲートウェイ発注（A社）」
- 監査証跡として何に使ったかを残す

#### 4. 適用日 *（必須）
- date input、デフォルトは当日
- 未来日は許容しない（過去日・当日のみ）

#### 5. メモ（任意）
- textarea、rows=2、placeholder「補足情報や関連資料のリンクなど」

### 更新プレビューカード（入力時にリアルタイム更新）
- グラデーション背景（blue 系薄塗り）で強調
- 「保存後の状態」を事前確認させる
- 表示内容:
  - 「今回 `+¥1,580,000`」
  - 「新実績 `¥5,780,000`」（= 現在実績 + 今回）
  - 「新残額 `¥4,220,000`」
- **Before / After バー**（2本縦並び）: 現在 42% → 保存後 57.8% のように色付きバーで視覚化
- 保存後に消費率の色分類（`.cr-safe/normal/warn/over`）が変わる場合は After バーの色が変化

### 閾値ガイダンス（info note）
- info アイコン + 薄青背景
- 「86% で注意、100% で超過アラートが全社に通知されます」
- 運用ルールを画面内に明示

### 過去の入力履歴（collapsible）
- 閉じた状態: 「過去の入力履歴（4件・合計 ¥4,200,000）」+ chevron-down
- 開いた状態: 直近4件を日付降順で表示
  - 各行: 日付 / カテゴリチップ / 金額（mono 右寄せ） / 用途
- `budget_actuals` テーブルから取得

### フッター
- **左**: 「この入力は変更履歴に記録されます」（muted、text-xs、info アイコン）
- **右**: 「キャンセル」（btn-outline）+ 「保存して反映」（btn-primary）
- 保存後: モーダル閉じる + 親画面（S-03c or S-04）をリフレッシュ + 成功トースト表示

## 設計上のこだわり（課題1/2 共通）

- **上書き方式から追加方式への段階的移行** — 課題1 では最小要件「案件単位の総額で可」を満たす上書き方式で実装。課題2 では監査証跡・内訳分析・誤入力耐性を獲得する追加方式に拡張する。DB の `projects.actual_amount` カラムは課題1/2 とも保持（課題2 では集計キャッシュとして扱う）ため、移行時の段階的マイグレーションが容易
- **「追加方式」が本来の理想形**（課題2 で実現） — 既存実績を上書きせず、`budget_actuals` テーブルに INSERT して積み上げる。監査証跡（いつ・誰が・何に使ったか）を残すため。上書き実装は実装コストは低いが、後から「何に使ったか」が追えなくなる
- **現状サマリー4カードを常時表示**（課題1/2 共通） — 入力前に「現在どうなっているか」を全画面でなく 4カードで即把握できる。フォームに入力しながら現状を参照でき、タブ切替や別画面を開く必要がない
- **Before / After プレビュー**（課題1/2 共通） — 保存前に「この入力が全体に与える影響」を視覚化。プログレスバーが変化するのを確認できる。「押したらどうなる分からない」不安を解消し、誤入力の抑止にもなる（桁を間違えると一気に 100% 超になるので気づける）
- **カテゴリチップを4色で分類**（課題2） — 外注費・ライセンス・機材費・その他。色を決めておくことで、過去履歴や S-03c の集計で「どこに一番使っているか」が視覚的に分かる
- **閾値ガイダンスを画面内に明示** — 86%/100% というルールを UI に書くことで、「なぜこの色なのか」を運用者全員が同じ基準で理解できる
- **主担当バッジをヘッダーに表示** — 誰が入力する画面か、また「自分が主担当なのか」を視覚化
- **過去の入力履歴を折り畳み配置**（課題2） — 参考情報なので閉じた状態がデフォルト。過去に何を計上したかと重複していないか確認したいときに使う
- **適用日を未来日不可**（課題2） — 「未来に使う予定」を実績として計上すると集計がずれる
- **保存後に成功トースト + 親画面リフレッシュ** — モーダル閉じた先で「保存された」ことを視覚的にフィードバック
