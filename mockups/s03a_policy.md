# S-03a 案件一覧（申請タブ） 方針

## 構成

- **背景**: ログイン直後の暫定ホーム画面。ダッシュボード（S-02）は課題2扱いのため、申請タブがランディング先
- **レイアウト**: AuthenticatedLayout（サイドバー「申請・承認 > 案件一覧」がアクティブ）、max-w-7xl
- **パンくず**: 申請・承認 > 案件一覧
- **URL**: `/projects?tab=approval`（承認待ちプリセットは `&filter=pending`）

## 構成セクション（上から下へ）

### ページヘッダー
- タイトル左に `.title-accent`（4px幅のアクセントカラー縦バー）
- 「案件一覧」+ サブタイトル「申請フェーズ · 全8件」
- 右端に「新規申請」ボタン（btn-primary、+アイコン）

### タブ切替UI（Tabs コンポーネント）
- 3タブ: 「申請」「開発」「予算」— アイコン + ラベル + 件数バッジ
- アクティブタブは jpt-red の下線3px + 赤文字
- アクティブタブの件数バッジは赤背景（`bg-[#FEE2E2] text-[#991B1B]`）、非アクティブは gray-100
- 右端に「フェーズ別に表示列が切り替わります」の補足テキスト（初回利用者向けガイダンス）
- タブクリックで URL クエリ `?tab=approval|dev|budget` を更新し、Inertia `router.visit()` で遷移

### フィルタバー
- 検索ボックス（w-80、左にSearchアイコン）: 案件名・申請者でテキスト検索
- ステータスセレクト: すべて / 下書き / 部門承認待ち / 本部承認待ち / 却下（approved は申請タブでは非表示選択肢）
- 部門セレクト: すべて / 開発1部 / 開発2部 / 開発3部
- 右端に「クリア」リンク（jpt-blue テキスト）
- フィルタはURLクエリパラメータで保持（`&status=pending_dept&department=1`）

### テーブル（ProjectTable: approval バリアント）
- 列構成: タイトル / ステータス / 承認ステップ / 申請日 / 部門 / 最終更新 / chevron
- タイトル列は2行構成 — 1行目 font-medium の案件名、2行目 text-xs muted の説明文
- rejected 行のタイトルに「再申請可」ミニバッジ（gray-100 背景）、説明文に却下理由の要約
- draft 行の申請日は「—」（未申請のため）
- 行全体が row-click（hover:bg-#F1F5F9、cursor-pointer）→ クリックで S-04 案件詳細へ
- 行末の chevron-right アイコン（muted）で「クリックで詳細に飛べる」ことを示唆

### 承認ステップ列（ApprovalStepperMini）
- 4ドット + 3ライン のインライン表示
- ドット状態: done（jpt-blue 塗り）/ current（jpt-blue 塗り + パルスアニメ）/ 未到達（白 + border）/ rejected（jpt-red 塗り）
- ライン状態: done（jpt-blue）/ 未到達（border色）
- ドット下にキャプション:「申請 → **部門** → 本部 → 済」— 現在位置を太字 + 色付き、rejected は「**部門で却下**」（赤）

### ページネーション
- 左に「8件中 1–8件を表示」
- 右にページ送りボタン（前へ / ページ番号 / 次へ）。現在ページは jpt-dark 塗り白文字
- Laravel の Paginator レスポンスに対応

### テーブル下凡例
- 5種の StatusPill を横並びで表示（draft / pending_dept / pending_hq / approved / rejected）
- 右端に「※ 承認済み案件は「開発タブ」で確認できます」— タブ切替の導線ヒント

## ロール別データアクセス（Controller 側クエリ分岐）

| ロール | 表示される案件 |
|---|---|
| applicant | `applicant_id = self` の案件 + 自分の draft |
| dept_manager | `department_id = self.dept` の全案件 |
| hq_manager | 全件 |

## 承認待ちプリセット（`/projects?tab=approval&filter=pending`）

サイドバー「承認待ち一覧」は独立画面を持たず、この画面のプリセット表示:
- dept_manager: `status = pending_dept` かつ `department_id = self.dept` で絞り込み
- hq_manager: `status = pending_hq` で全部門を表示
- applicant: このメニューは非表示（または自分の pending 案件のみ）
- UI は S-03a と完全共通。タイトル下に「承認待ち」バッジで文脈を示す

## 設計上のこだわり

- **「同一コンポーネント + タブ切替」で3画面を1つに集約** — 申請・開発・予算の3タブは同じ `Projects/Index.tsx` で列セットだけ切り替える。Controller は `?tab` パラメータで返す列（eager load するリレーション）を変える。別画面に分割するよりルーティングが単純で、ユーザーもタブクリック1つでフェーズ横断できる
- **承認ステッパーMini は「今どこにいるか」の一覧レベル要約** — S-04 の Full ステッパーと同じデータを 10px のドットに圧縮。一覧では詳細は不要で、「何ステップ進んでいるか」がパッと見える粒度が正解。キャプション行で現在位置を文字でも補足し、ドットだけでは読み取れないユーザーにも対応
- **フィルタはURLクエリで永続化** — ブラウザバックやブックマークでフィルタ状態を復元できる。Inertia の `router.visit()` で URL を書き換え、Controller 側で `request()->query()` から分岐。フロントの状態管理は不要
- **rejected 行に「再申請可」バッジと却下理由サマリ** — 却下された案件が「もう終わり」なのか「修正して出し直せる」のかを一覧レベルで判別可能にした。再申請は `parent_project_id` で元案件に紐づく新規レコードとして作成
- **アクティブタブの件数バッジを赤背景にする** — 非アクティブタブの灰色バッジとの視覚的コントラストで「今見ているタブ」を強調。件数自体がアクション喚起になる（pending が多い → 対応が必要）
- **凡例を最下部に配置** — 初回利用時にステータスの色と意味を確認できる。「承認済みは開発タブで」のヒントでタブの使い分けも誘導
- **アクセントカラーはタイトル縦バーのみ** — S-04 と同じ `.title-accent` の4px バーで統一。テーブル内やタブには使わず、ブルー系（ステッパー・リンク）とレッド系（CTA・アクティブタブ）で情報を整理
