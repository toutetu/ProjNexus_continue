# S-03b 案件一覧（開発タブ） 方針

## 構成

- **背景**: 承認済み案件の開発進捗を俯瞰する画面。サイドバー「開発管理 > 案件一覧」から遷移
- **レイアウト**: AuthenticatedLayout（サイドバー「開発管理 > 案件一覧」がアクティブ）、max-w-7xl
- **パンくず**: 開発管理 > 案件一覧
- **URL**: `/projects?tab=dev`
- **コンポーネント**: S-03a / S-03c と同じ `Projects/Index.tsx` を tab props で列切替

## 構成セクション（上から下へ）

### ページヘッダー
- タイトル左に `.title-accent`（4px幅のアクセントカラー縦バー）
- 「案件一覧」+ サブタイトル「開発フェーズ · 全5件」
- 右端に「新規申請」ボタン（btn-primary、+アイコン）※ 開発タブでも CTA は統一

### タブ切替UI（Tabs コンポーネント・dev がアクティブ）
- 3タブ共通。dev タブの件数バッジが赤背景（`bg-[#FEE2E2] text-[#991B1B]`）で強調
- 他タブ（申請・予算）は gray-100 バッジ
- タブクリックで URL クエリ `?tab=approval|dev|budget` を更新

### フィルタバー
- 検索ボックス（w-80）: 案件名・主担当名でテキスト検索
- 部門セレクト: すべて / 開発1部 / 開発2部 / 開発3部
- 進捗セレクト（開発タブ固有）: すべて / 未着手(0%) / 進行中 / 完了間近(90%+) / 完了
- 右端に「クリア」リンク
- ※ ステータスフィルタは **なし**（dev タブは `status=approved` で Controller 側固定）

### テーブル（ProjectTableDevTab）
- 列構成: タイトル / 部門 / 主担当 / タスク進捗 / 期限 / 最終更新 / chevron
- **タイトル列**: 2行構成 — 案件名（font-medium）+ PRJ-ID（mono、muted、例「PRJ-2026-0042」）
- **主担当列**: avatar（イニシャル円、グラデーション背景） + 氏名
- **タスク進捗列**:
  - プログレスバー（w-32, h-2）+ パーセンテージ（右側 mono、w-10）
  - バー色: 通常=`var(--jpt-blue)`、95%以上=`#16A34A`（緑、完了間近）
  - 進捗率 = `tasks.where(status=done).count() / tasks.count() * 100`
- **期限列**: 日付表示 + 残日数ラベル
  - `.dl-soon`: 2週間以内 → 赤文字 + 赤背景ラベル
  - `.dl-warn`: 1ヶ月以内 → オレンジ文字
  - それ以上: muted
- **完了間近バッジ**（`.pill-completing`、紫）: 進捗 90% 以上の行のタイトル横に表示
- 行全体が row-click → S-04 案件詳細へ

### ページネーション
- S-03a と同じ構成（左に件数表示、右にページ送り）

## ロール別データアクセス（Controller 側クエリ分岐）

| ロール | 表示される案件 |
|---|---|
| applicant | `status=approved` かつ `primary_assignee_id = self` の案件 |
| dept_manager | `status=approved` かつ `department_id = self.dept` の全案件 |
| hq_manager | `status=approved` の全件 |

※ 申請タブとの違い: 開発タブは承認済のみ。却下・下書きは表示しない

## 設計上のこだわり

- **ステータスフィルタを UI から削除** — 開発タブは `status=approved` で固定されるため、ステータス選択肢を出すと混乱を招く。代わりに進捗フィルタ（未着手/進行中/完了間近/完了）を用意し、「開発の段階」で絞り込める軸に置き換えた
- **主担当 avatar を一覧で表示** — 開発フェーズで最も大事なのは「今誰が動かしているか」。テキストだけでなく avatar で視覚化することで、複数案件を横断しても担当者を瞬時に識別できる
- **タスク進捗バーを集計値で表示** — `projects` テーブルには進捗カラムを持たず、`tasks` の集計で算出する（DB に冗長カラムを持たない方針）。Eloquent の `withCount` と `tasks.status=done` の条件集計で実装
- **進捗 95% 以上でバーを緑に変える** — 「あと一息」を視覚的にフィードバックして最後のモチベーション維持につなげる。100% だけを緑にすると「まだ青」の状態が長く続いて達成感が出ないため、95% 以上で先行的に色を変える
- **期限色分けで「やばい案件」を抽出** — ダッシュボード（S-02）がなくても、この一覧だけで「対応が遅れている案件」を見つけられるようにする。2週間以内=赤、1ヶ月以内=橙、それ以降=muted の3段階
- **完了間近バッジ（紫）** — 他の色（赤=警告、緑=完了、青=進行中）と被らない紫をポジティブイベント（もうすぐゴール）に当てた。色の役割重複を避けつつ、ゴール間近の高揚感を表現
- **PRJ-ID を一覧でも表示** — Slack やメールで「PRJ-2026-0042 の件」と参照されることを想定。ID ベースで案件を探せる運用を支援
- **同一コンポーネント + タブ切替** — S-03a / S-03b / S-03c は `Projects/Index.tsx` 1ファイル。Controller 側で `?tab` を見て返す Eloquent eager load を変え、フロント側は tab props で列セットを切替。別画面に分けるよりルーティングが単純
