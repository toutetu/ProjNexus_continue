# S-04 案件詳細 方針

## 構成

- **背景**: アプリ全体で最も情報量の多い画面。案件一覧（申請/開発/予算タブ）や承認待ち一覧の行クリックで遷移。3セクションすべてからの合流点
- **レイアウト**: AuthenticatedLayout（サイドバー「開発管理 > 案件一覧」がアクティブ）、max-w-7xl
- **パンくず**: 開発管理 > 案件一覧 > {案件タイトル}
- **表示例のステータス**: `pending_hq`（本部承認待ち）— ステッパーが最も情報量のある状態

## 構成セクション（上から下へ）

### ページヘッダー
- タイトルの左に `.title-accent`（4px幅のアクセントカラー縦バー）
- タイトル横にステータスバッジ（StatusPill）
- サブ情報行: 部門アイコン + 部門名 / ユーザーアイコン + 申請者名 / カレンダーアイコン + 起票日
- 右側にコンテキスト操作ボタン: 「編集」（draft時のみ）/「再申請」（rejected時のみ）— 今回の pending_hq 状態ではいずれも非表示


### 2段目: タブナビ（DetailTabs）
- 4タブ: 「概要」「タスク（件数バッジ）」「予算」「履歴」
- 「履歴」タブは内部で **承認履歴 / 変更履歴** のサブセクションに分かれる（タイムラインを2本）
- アクティブ色は jpt-red + 下線3px（一覧タブと統一）
- タブ切替で下部コンテンツが差し替わる
- URL クエリ `?detailTab=overview|tasks|budget|history` で初期タブを指定可能

> **補足**: 初期モック作成時にタブを「概要 / タスク / 承認履歴 / 変更履歴」としていたが、一覧側（S-03c 予算タブ）からの流入を想定すると「予算」タブが独立している方が動線が自然なため、`申請 / 履歴　/ タスク / 予算 ` に正したうえで履歴タブの中で承認履歴・変更履歴を並列表示する構成に修正した。

### 申請タブ（grid-cols-1 lg:grid-cols-3）

####  承認ステッパーUI（ApprovalStepperFull）
- カード内に4ステップを横配置: [申請] → [部門承認] → [本部承認] → [承認済]
- 完了ステップ: 塗りつぶし円（jpt-blue）+ ✓アイコン + 担当者名・日時を下に表示
- 進行中ステップ: パルスアニメ円（jpt-blue）+ clockアイコン + 「対応待ち」+ 担当者名
- 未到達ステップ: 枠線のみのグレー円 + 番号
- コネクタ線: 完了区間は jpt-blue、未到達区間は border 色
- カードヘッダー: muted アイコン + 「承認フロー」+ 右端に申請日

**内容は、申請画面と同じ**
## 内蔵モーダル: S-08 承認/却下（ApprovalDialog）
- max-w-lg、中央配置、背景 dimmed
- **承認モード**: 緑アイコン + 「案件を承認」タイトル + 実行される処理一覧（緑の info box）+ コメント欄（任意）+ 「承認する」(btn-green)
- **却下モード**: 赤アイコン + 「案件を却下」タイトル + 実行される処理一覧（赤の info box）+ 却下理由欄（必須）+ 「却下する」(btn-primary=赤)
- Esc / backdrop クリックで閉じる


- **関連情報カード**: 案件ID（mono）/ 申請回数 / 主担当


### 履歴タブ（承認履歴 + 変更履歴）
- 1タブ内に 以下の内容を併せて、時系列に表示（`max-w-2xl` 左寄せ）

**承認履歴（Timeline 形式）**
- 各エントリ: 色付きタイムラインドット + アイコン + 「○○さんが{アクション}しました」+ 日時 + コメント（灰色カード内）
- ドット色: 申請=blue、承認=green、待機中=blue
- 最終エントリが待機中なら「本部承認を待っています」+ 承認者名

**変更履歴（Timeline 形式）**
- 同じ Timeline コンポーネントを使用
- ステータス変更は StatusPill の before → after で可視化（「下書き → 部門承認待ち」など）
- 数値変更は取り消し線 → 新値で表現（「¥8,000,000 → ¥10,000,000」）

### タスクタブ（status=approved 時のみ実質稼働）
- ヘッダー: 「タスク一覧」+ 件数バッジ + 「タスク追加」ボタン（btn-primary）
- テーブル列: タイトル / 種類バッジ(bug/feature/improve/task) / 優先度バッジ(高/中/低) / ステータス / 担当 / 期日 / 進捗バー+%
- 行クリック → S-10 タスク編集モーダル
- フッター: 「5件表示 / 全12件」+ 「すべてのタスクを表示 →」リンク

### 予算タブ
- S-03c のサマリーカード + 予算実績履歴 + 「予算実績入力」ボタン（主担当のみ）で構成
- 上段: 確定予算 / 実績合計 / 残額 / 消費率（ProgressBar 4段階色分け）の4カード
- 中段: `budget_actuals` の履歴テーブル（日付 / カテゴリチップ / 金額 / 用途 / 入力者）
- 下段: 「予算実績入力」ボタン（btn-primary、coins アイコン）→ S-11 モーダルを開く
- 未承認時（`status != approved`）はタブ自体は表示するが、入力ボタンは disabled


- **予算サマリーカード**: 概算予算・確定予算・実績額の3カラム数値表示 + 消費率プログレスバー。未承認時は確定予算「—」、実績「¥0」、消費率「0%（未承認）」

- **予算実績入力カード**（主担当のみ・承認後に解禁）: 未承認時は opacity-50 + disabled で「承認後に実績額の入力が可能になります」



## 設計上のこだわり

- **ステッパーは承認フローの「可視化」** — pending_hq では3ステップ目がパルスし、誰が次のアクションを持っているか一目でわかる。これは課題2要素だが、低コスト高効果なので課題1に含めて実装する
- **右カラムのアクションは「今の自分に何ができるか」** — ロール×ステータスで表示/非表示が切り替わり、不要なボタンは見せない（認知負荷の削減）。予算実績入力は承認前なら disabled + 説明文で「なぜ押せないか」を伝える
- **タブで情報階層を整理** — 概要（日常的に見る）→ タスク（開発フェーズ）→ 承認履歴（監査証跡）→ 変更履歴（トレーサビリティ）の頻度順
- **承認モーダルに「実行される処理一覧」を表示** — 承認ボタンを押すと何が起きるか（予算転記、通知発行など）を事前に提示。「承認したら何が変わるか分からない」不安を解消する
- **アクセントカラー(#EDB100)はタイトル左の縦バーに限定** — ページ内の色をブルー系（ステッパー・進捗バー・リンク）とレッド系（CTA・タブ）に統一し、アクセントはさりげないポイント使いに留めた
- **ロゴのグラデーションはシアン→ブルー→パープル** — サイドバー全画面で共通のブランドカラーとして一貫させる
